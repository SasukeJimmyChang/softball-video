# 伺服器端骨架偵測架構計畫

## 為什麼需要伺服器端骨架偵測

目前的做法是純靠 Gemini Vision「看」圖片分析姿勢，問題：
- 無法精確量化角度（只能目測估算）
- 每次分析結果不一致（LLM 隨機性）
- 看不清的地方只能猜測

加入伺服器端骨架偵測後：
- MediaPipe 計算出 33 個關節的精確 3D 座標
- 可以精確計算所有角度（肘角、膝角、髖肩分離角等）
- 角度數據 + Gemini Vision 雙重驗證 → 大幅提升準確度和一致性

## 架構設計

```
[手機/電腦]                    [Linode VPS]
    │                              │
    │ 上傳影片(FormData)            │
    │──────────────────────────────>│
    │                              │
    │                    ┌─────────┴──────────┐
    │                    │   Python 後端       │
    │                    │   (FastAPI/Flask)   │
    │                    │                     │
    │                    │  1. MediaPipe Pose  │
    │                    │     → 33 landmarks  │
    │                    │     → 計算角度      │
    │                    │                     │
    │                    │  2. 擷取關鍵幀      │
    │                    │     (OpenCV)        │
    │                    │                     │
    │                    │  3. 呼叫 Gemini API │
    │                    │     (角度數據+圖片)  │
    │                    │                     │
    │                    └─────────┬──────────┘
    │                              │
    │ 回傳分析報告(JSON)            │
    │<─────────────────────────────│
```

## 技術選型

| 元件 | 選擇 | 原因 |
|------|------|------|
| 伺服器 | Linode VPS ($5-10/月) | 無執行時間限制，可安裝 Python |
| 後端框架 | FastAPI (Python) | 非同步、快速、支援 file upload |
| 骨架偵測 | MediaPipe Pose (Python) | 免費、33 landmarks、3D 座標 |
| 影片處理 | OpenCV (Python) | 擷取幀、縮放、格式轉換 |
| AI 分析 | Gemini API | 結合角度數據 + 視覺分析 |
| 前端 | Next.js（現有） | 不需大改，改 API 端點即可 |
| 部署 | Docker + Nginx | 一鍵部署，SSL 自動化 |

## 角度計算清單

伺服器端 MediaPipe 可精確計算的角度：

### 打擊
| 項目 | 計算方式 |
|------|---------|
| B-03 後手肘角度 | shoulder-elbow-wrist 三點角度 |
| B-04 前手肘角度 | shoulder-elbow-wrist 三點角度 |
| B-05 站距 | 兩腳踝距離 / 兩肩距離 |
| B-06 前膝角度 | hip-knee-ankle 三點角度 |
| B-08 肩線傾斜 | 兩肩連線與水平線的夾角 |
| B-10 髖部旋轉 | 兩髖連線角度的逐幀變化 |
| B-13 踏步距離 | 前腳移動距離 / 肩寬 |
| B-15 脊椎側傾 | 肩中點-髖中點連線與垂直線的夾角 |
| B-16 前手延伸 | 肩-肘-腕 展開角度 |
| B-18 後膝 Valgus | 膝蓋相對於髖-踝連線的偏移量 |

### 投球
| 項目 | 計算方式 |
|------|---------|
| P-01 肘部高度 | 肘 Y 座標 vs 肩 Y 座標 |
| P-03 肘角度 | shoulder-elbow-wrist 三點角度 |
| P-04 肩線傾斜 | 兩肩連線與水平線夾角 |
| P-08 跨步距離 | 後腳到前腳的距離 / 身高估算 |
| P-12 髖肩分離 | 兩髖連線角度 - 兩肩連線角度 |
| P-13 軀幹前傾 | 肩中點-髖中點 vs 垂直線 |

## 新的分析流程

```
1. 上傳影片 → 伺服器接收
2. OpenCV 擷取幀（每 0.1 秒）
3. MediaPipe 偵測每幀的 33 個 landmarks
4. 計算所有角度並生成角度時序數據
5. 選取關鍵幀圖片（高品質，不需壓縮）
6. 組合 prompt：角度數據表 + 圖片 → Gemini
7. Gemini 結合精確角度 + 視覺判斷 → 報告
8. 回傳 JSON 報告給前端
```

## 預期效果提升

| 項目 | 目前（純 Gemini Vision） | 改進後（MediaPipe + Gemini） |
|------|------------------------|--------------------------|
| 角度精度 | ±20-30°（目測） | ±2-5°（計算） |
| 評分一致性 | 每次可能不同 | 角度固定 → 評分穩定 |
| 大檔案支援 | 受限 4.5MB | 無限制（伺服器處理） |
| 骨架顯示 | 無 | 可在影片上疊加骨架 |
